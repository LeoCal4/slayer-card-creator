Intro What does Slayer Card Creator do?

What is this app?

  • Design card layouts on a visual canvas ‚Äî add layers (backgrounds, text, art, badges) and arrange them like a mini card-game Photoshop
  • Manage your card list ‚Äî import hundreds of cards from a CSV file, or add and edit them manually one by one
  • Preview every card rendered through its template in a live, scrollable grid
  • Export to Cockatrice ‚Äî one click produces an XML card database + one PNG per card, all packed into a ZIP
  • Works entirely offline. No accounts, no internet, no cloud.

Cards Adding and editing cards

Card List ‚Äî Adding & Editing

  • Click Add Card to create a new blank row ‚Äî fill in fields directly in the table
  • Set the Type first ‚Äî it determines which fields are required and which are greyed out
  • Required fields left blank get a red outline as a reminder
  • Power, HP, and VP cells are disabled for types that don't use them
  • Use the search box to filter by name or effect; click column headers to sort
  • Click the delete icon on any row to remove a card (confirmation required)

Cards The nine card types and their required fields

Card Types & Required Fields

Type Required fields
Slayer name, class, type, rarity, cost, power, hp, effect
Errant name, class, type, rarity, cost, power, hp, vp, effect
Action / Ploy / Intervention name, class, type, rarity, cost, effect
Chamber / Relic name, class, type, rarity, cost, effect
Dungeon / Phase name, type, effect (no class, rarity, or cost)
Rarity values
Valid: common, rare, epic. A blank rarity defaults to common. Multi-class cards: separate class names with a /.
Set Info Your set's identity

Set Info ‚Äî Your Set's Identity

  • Set Name ‚Äî the display name shown in Cockatrice
  • Set Code ‚Äî up to 8 characters, auto-uppercased; used as the filename inside the exported ZIP (e.g. SLYR ‚Üí SLYR.xml)
  • Release Date ‚Äî recorded in the Cockatrice XML database
  • Art Folder ‚Äî click Select Art Folder to point at the folder holding your card art images; files should be named to match card names
Art file naming
If an art file isn't found for a card, the export uses a grey placeholder with the card name. Missing art is reported as a warning in the Export view ‚Äî not an error.
Set Info Class colors & phase configuration

Class Colors & Phases

Class Palette
  • One row per class ‚Äî set the name, primary, and secondary color via the color picker or hex input
  • These colors are referenced by Rect layers in the Template Designer using class.primary or class.secondary fill sources
  • Click Add Class to add a new class; use the delete icon to remove one
Phase Configuration
  • Four phases: Encounter, Preparation, Combat, Camp
  • Set a single-letter abbreviation for each (shown on Phase Icon layers)
  • The checkbox grid assigns which card types play in which phases ‚Äî drives the Phase Icons layer in the Template Designer
Templates The visual canvas editor

Template Designer ‚Äî Overview

  • Each template is bound to one or more card types ‚Äî Slayer and Errant can share one template if they look the same
  • A template is a stack of layers rendered bottom-to-top ‚Äî exactly like any design tool
  • The Layer Panel (left side) lists all layers; click one to select it, or use the visibility and lock icons
  • The Properties Panel (right side) shows all editable settings for the selected layer
  • The Preview Card Selector above the canvas lets you see a real card from your list rendered live as you edit
Starter templates
New projects include four ready-made templates. Feel free to edit, rename, or delete them.
Templates The five layer types

Layer Types

Rect
Solid fill or class-colour fill (primary / secondary / gradient). The building block for card frames and colour zones.
Text
Displays any card field ‚Äî name, effect, class, rarity, and more. Set font, size, alignment, and colour.
Image
Card art loaded from the art folder, or a frame image you upload directly into the template (stored as base64).
Badge
Circular label for a numeric field ‚Äî cost, power, HP, or VP. Fully customisable fill and text colour.
Phase Icons
Renders all phases assigned to a card's type as small icon chips using the abbreviations set in Set Info.
Show If Field
Every layer supports a Show If Field condition ‚Äî the layer is only rendered when the chosen card field has a value. Great for optional fields like VP.
Templates Tips & useful features

Designer Tips

  • Drag any layer on the canvas to move it; grab the corner handles to resize it
  • Enable Snap-to-Grid (toggle above the canvas) for pixel-perfect placement ‚Äî choose a 1, 5, or 10 px grid size
  • Use the canvas width √ó height inputs to set a custom card size ‚Äî the default is 375 √ó 523 px
  • Use Export Template to save a template as its own JSON file, and Import Template to bring it into another project
  • Lock a layer in the Layer Panel to prevent accidental moves while you work on other layers
Preview Seeing all your cards at once

Preview Grid

  • The Preview view renders every card through its matched template in a responsive grid
  • Cards are lazy-loaded ‚Äî rendering starts as each tile scrolls into view, so large sets don't freeze the app
  • Use the type filter dropdown to narrow the view to one card type at a time
  • Click Render All to batch-render every card in sequence ‚Äî a counter shows progress
  • If a card has no matching template it won't appear ‚Äî a warning will show in the Export view
Tip
Use Preview to catch layout issues before exporting. Changes made in the Template Designer take effect immediately when you re-render.
Export Generating your Cockatrice package

Exporting to Cockatrice

  1. 1
    Open the Export view ‚Äî check the summary for missing templates or art files before continuing
  2. 2
    Click Export ZIP ‚Äî the app renders every card and reports progress as it goes
  3. 3
    A save dialog appears ‚Äî pick a location. The ZIP contains <code>.xml and pics/CUSTOM/<name>.png for every card
  4. 4
    In Cockatrice: Card Database ‚Üí Import set from ZIP ‚Äî your cards appear immediately
What's in the ZIP
The XML uses Cockatrice format version 4. Card images are PNGs at whatever canvas size your template uses (default 375 √ó 523 px).
Projects Saving and loading your work

Saving & Project Files

  • Press Ctrl+S to save ‚Äî a dialog appears on first save to pick a filename and location
  • A ‚óè next to the project name in the header means there are unsaved changes
  • Projects are plain .json files ‚Äî all cards, templates, set info, and class colors are stored inside
  • Frame images uploaded to templates are embedded as base64 ‚Äî no external files needed
  • Card art is stored as a folder path and loaded at runtime ‚Äî keep the folder in the same location on disk
  • Closing the window with unsaved changes shows a confirmation dialog ‚Äî you won't lose work by accident
